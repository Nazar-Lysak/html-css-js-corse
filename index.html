Домашнє завдання № 2

1) Зробити HTML-розмітку персонального проекту аналогічно тому, як ми це робили на занятті. Це буде один і той самий код, що для мобільної, що для десктопної версії, відрізнятися вони будуть лише CSS. Код має бути добре відформатованим, проходити валідатор, використовувати семантичні теги, класи, і айді мають бути гарно названі. Зараз робіть без стилів CSS, лише HTML.

Зараз ми фокусуємося виключно на HTML - це лише теги з інформацією, зовнішній вигляд нас не цікавить. CSS не додавайте взагалі, щоб вас не відволікало, як це робити правильно, почнемо вчити з наступного заняття. Коли пишeте HTML-код постарайтеся особливу увагу приділити семантичним тегам - header, footer, article і т.д. :point_right:Сайт має проходити HTML-валідатор, у якості виключення допускаються зауваження стосовно застосування CSS - це поки можете ігнорувати. Всі інші недоліки у валідаторі необхідно усунути. Якщо ви не розумієте, що від вас хоче валідатор - закидуйте код на гітхаб, робіть скріншот з повідомлення валідатора та надсилайте питання в групу.

У консолі та вкладці Network не має бути помилок, зокрема 404.

Слідкуйте за назвами файлів. Перша сторінка сайту - завжди index.html, назви файлів і папок робіть виключно маленькими латинськими літерами і з'єднуйте слова дефісами, наприклад так: "grumpy-cat.jpg". Не використовуйте цифри в іменах файлів, наприклад "img1.jpg" - то дуже погана назва. З назви файлу має бути зрозуміло, що в ньому. Для картинок створюйте окрему папку "img" та складайте картинки до неї.

Код має бути добре відформатований, для цього в VS Code та Gitpod.io використовуйте комбінацію клавіш Alt-Shift-F, можна ще використовувати кнопочку форматування в браузері, яка доступна в Developer Tools, Sources.

Під час створення форм особливу уваги приділіть тому, щоб правильно приєднати label до input. Перевірити правильність приєднання дуже просто - достатньо клікнути по label, щоб активувався відповідний input.

Результат, який отримаєте в кінці, має бути обов’язково завантажений на GitHub.
